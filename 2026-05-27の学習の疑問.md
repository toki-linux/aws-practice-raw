# AWSネットワーク学習メモまとめ

## ルートテーブル

ルートテーブルでは、宛先に対して **より具体的なルートが優先** される。

例：

```text
10.0.0.0/16 → local
0.0.0.0/0  → Internet Gateway
```
宛先が 10.0.2.10 の場合、10.0.0.0/16 に含まれるため、local が使われる。

この仕組みを 最長プレフィックス一致 という。

より細かく指定されているルートが優先される

0.0.0.0/0 は、どれにも当てはまらなかった通信の最終出口のようなもの。

## サブネットとルートテーブル

1つのサブネットが使えるルートテーブルは 1つだけ。

ただし、1つのルートテーブルの中には複数のルートを書ける。

サブネット = 1冊の地図だけ持てる
ルートテーブル = 地図
ルート = 地図の中に書かれた道案内
## ルート伝播

ルート伝播とは、VPN Gatewayなどから受け取った経路情報を、ルートテーブルへ自動追加する仕組み。

手動でルートを書く
→ 今やっている基本

ルート伝播
→ VPNやDirect Connectなどで使う応用
default Security Group

default セキュリティグループは、VPCごとに1つ自動で作られる。

そのため、default が2つある場合は、基本的にVPCが2つあるということ。

VPC 1 → default SG
VPC 2 → default SG
## Security GroupのSource指定

Security GroupのSourceには大きく2種類ある。

1. CIDR指定
   例: My IP、10.0.1.0/24、10.0.1.10/32

2. Security Group参照
   例: public-ec2-sg

この2つは別物。

エラーの原因は、IPv4 CIDRを書く欄にSG IDを入れようとしたこと。

CIDRで許可するルール
SGで許可するルール
→ 別枠

SG参照として作れば、CIDRは不要。

## プレフィックスリスト

プレフィックスリストは、CIDRの詰め合わせセット。

複数のCIDRをまとめたリスト

毎回CIDRを直接書く代わりに、

このリストに入っているIP範囲を許可する

または、

このリストに入っている宛先へはこのルートを使う

という形で使う。

## .pemファイルの権限確認

ホームディレクトリ直下にある .pem ファイルの権限を見るコマンド。

ls -l ~/*.pem

意味：

ls -l  → 権限などの詳細表示
~      → ホームディレクトリ
*.pem  → .pemで終わるファイル
My IPとは

AWSコンソールの My IP は、自分のPC側のグローバルIP。

例：

203.xxx.xxx.xxx/32

/32 は、そのIPアドレス1個だけを意味する。

## My IPとEC2のPublic IPの違い
自分のPCのグローバルIP
→ Security GroupのSourceに使う

public-ec2のPublic IP
→ SSHコマンドの接続先に使う

例：

ssh -i key.pem ubuntu@public-ec2のPublicIP

このときの通信は以下。

自分のPC
  送信元IP: My IP
        ↓
public-ec2
  宛先IP: public-ec2のPublic IP

つまり、

My IP = 接続してくる側
public-ec2のPublic IP = 接続される側の宛先
