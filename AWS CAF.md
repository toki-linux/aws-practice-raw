# AWS CAF 6つのパースペクティブと各ケイパビリティの簡単な意味

## AWS CAFとは

AWS Cloud Adoption Framework（AWS CAF）は、組織がクラウド導入やクラウドを活用した変革を進めるための考え方を整理したフレームワーク。

クラウド導入では、技術だけでなく、経営、人材、ルール、セキュリティ、日々の運用まで考える必要がある。

AWS CAFでは、確認する領域を次の6つのパースペクティブに分けている。

```text
ビジネス
人材
ガバナンス
プラットフォーム
セキュリティ
オペレーション
```
6つのパースペクティブの最短整理
```txt
ビジネス
＝クラウドでどのような事業成果を得るか

人材
＝組織、人、文化をどう変えるか

ガバナンス
＝クラウド導入をどのようなルールで管理するか

プラットフォーム
＝AWS上にどのような技術基盤を作るか

セキュリティ
＝ID、データ、システムをどう守るか

オペレーション
＝構築後のシステムをどう監視・運用するか
```
1. ビジネスのパースペクティブ

ビジネスのパースペクティブでは、クラウドを導入することで、どのような事業成果を得るかを考える。

キーワード
＝経営戦略、製品、利益、顧客、イノベーション、データ活用
ストラテジーマネジメント

組織の経営戦略とクラウド活用を結び付ける。

クラウドを使って、
どのような事業目標を達成するかを決める

例：

新しい市場へ進出する
サービス提供を高速化する
顧客満足度を向上させる
ポートフォリオマネジメント

複数の事業、製品、プロジェクトの中から、優先して投資する対象を決める。

何からクラウド化するか
どの取り組みを優先するか
イノベーションマネジメント

新しい製品やサービスを生み出したり、既存の業務を改善したりする。

新しいアイデアを試し、
価値のあるものを事業化する
プロダクトマネジメント

製品やサービスを、企画から開発、提供、改善、終了まで継続的に管理する。

製品のライフサイクル全体を管理する
戦略的パートナーシップ

AWS、AWSパートナー、取引先などと協力し、クラウド導入を進める。

自社だけですべて行わず、
外部の専門家や企業と協力する
データ収益化

保有しているデータを利用して、新しい売上や事業価値を生み出す。

データを利益につなげる

例：

顧客データを使った新サービス
分析結果を有料で提供
データを使った広告の最適化
ビジネスインサイト

データを分析して、経営や業務の判断に役立つ情報を得る。

データから、
現在の状況や次に取るべき行動を知る
データサイエンス

統計、実験、分析、機械学習などを使って、ビジネス上の問題を解決する。

データを使って予測・分類・最適化を行う

例：

売上予測
解約予測
不正利用の検出
商品の推薦
ビジネスの覚え方
クラウドを使って
何を実現し、どのように利益を得るか
→ ビジネス
2. 人材のパースペクティブ

人材のパースペクティブでは、クラウド導入に対応できるように、人、組織、文化、働き方を変える。

キーワード
＝人材育成、組織文化、リーダー、役割、変革
文化の進化

現在の組織文化を確認し、クラウドに適した文化へ変えていく。

失敗を恐れず試す
部門を越えて協力する
継続的に改善する
トランスフォーメーションリーダーシップ

経営者やリーダーが、変革の目的や方向性を示し、組織を導く。

リーダーがクラウド変革の方針を示す

単に優先順位を決めるだけでなく、組織全体へ変革の必要性を伝え、意思決定を行う。

クラウドフルエンシー

経営者、技術者、業務担当者が、それぞれの役割に必要なクラウドの知識を持つ。

組織全体のクラウド理解力を高める

技術者だけがAWSサービスを学ぶという意味ではない。

ワークフォースのトランスフォーメーション

クラウド導入に必要な人材を育成し、役割やスキルを変えていく。

研修
再教育
採用
新しい職種や役割の設置
変革の促進

クラウド導入によって影響を受ける人や業務を特定し、混乱を抑えながら変革を進める。

変更を組織へ定着させる

例：

新しい業務手順を説明する
社員の不安を減らす
導入状況を確認する
組織設計

クラウドを効果的に利用できるように、組織構造、役割、責任を見直す。

誰が何を担当するかを整理する
組織連携

経営、開発、運用、セキュリティなどの部門が、継続的に協力できる関係を作る。

部門間の壁を減らして協力する
人材の覚え方
クラウドを使える人と組織を作る
→ 人材
3. ガバナンスのパースペクティブ

ガバナンスのパースペクティブでは、クラウド導入を組織の方針やルールに従って管理する。

クラウド導入プロジェクトを予定どおり進める
利益管理

例：

開発期間短縮
売上増加
顧客満足度向上

クラウド導入で発生する可能性のあるリスクを特定し、対策する。

発生確率や被害を小さくする
クラウド財務管理

クラウドの利用料金を把握し、予算、支出、コスト配分を管理する。

AWS料金を見える化し、無駄を減らす

FinOpsに関係する領域。

アプリケーションポートフォリオ管理

組織が所有するアプリケーションを一覧化し、維持、移行、廃止などを判断する。

どのアプリを残すか
どのアプリをクラウドへ移すか
どのアプリを廃止するか
データガバナンス

データを誰が所有し、誰が利用でき、どのように保護・管理するかを決める。

データに関するルールと責任を決める

例：

アクセス権限
保存期間
データ分類
個人情報の扱い
データキュレーション

データを収集、整理、分類、品質確認し、利用しやすい状態にする。

バラバラなデータを、
信頼して使える状態に整える
ガバナンスの覚え方
クラウドを勝手に使うのではなく、
ルールと計画に沿って管理する
→ ガバナンス
