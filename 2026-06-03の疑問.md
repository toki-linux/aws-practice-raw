
# AWSネットワーク・データベース関連用語の整理

## RDSとは？

### 概要

Amazon RDS（Amazon Relational Database Service）は、AWSが提供するリレーショナルデータベースのマネージドサービスである。

一言で表すと、以下のようなサービス。

```text
RDS
↓
データベースを簡単に構築・運用するためのAWSサービス
↓
利用するデータベースエンジンを選択する
```

RDSそのものが、MySQLという種類のデータベースなのではない。

RDSは、データベースを作成・管理するためのサービス。

その中で、実際に利用するデータベースエンジンを選ぶ。

---

## データベースエンジンとは？

データベースエンジンは、データを保存・検索・更新するために実際に動作するソフトウェアである。

RDSでは、用途に応じて複数のデータベースエンジンから選択できる。

代表例は以下のとおり。

| データベースエンジン | 特徴 |
|---|---|
| MySQL | 広く利用されているオープンソースのリレーショナルデータベース |
| PostgreSQL | 高機能なオープンソースのリレーショナルデータベース |
| MariaDB | MySQLから派生したオープンソースのリレーショナルデータベース |
| Oracle Database | Oracle社が提供する商用データベース |
| Microsoft SQL Server | Microsoft社が提供する商用データベース |
| Amazon Aurora | AWSがクラウド向けに設計したMySQL・PostgreSQL互換のデータベース |

イメージは以下のとおり。

```text
Amazon RDS
├── MySQL
├── PostgreSQL
├── MariaDB
├── Oracle Database
├── Microsoft SQL Server
└── Amazon Aurora
```

---

## 今回の実践で利用したもの

今回の実践では、RDSのデータベースエンジンとしてMySQLを選択した。

```text
Amazon RDS
↓
データベースエンジンとしてMySQLを選択
↓
RDS for MySQL
```

Private EC2からは、RDS EndpointへTCP 3306で接続した。

```text
Private EC2
↓ TCP 3306
RDS Endpoint
↓
RDS for MySQL
```

---

## VPC Flow Logsとは？

### 概要

VPC Flow Logsは、VPC内で発生したIP通信に関する情報を記録する機能である。

一言で表すと、以下のような機能。

```text
VPC Flow Logs
=
VPC内のIP通信ログを記録する機能
```

---

## 何を確認できるのか

VPC Flow Logsでは、通信が許可されたか、拒否されたかを確認できる。

たとえば、以下のような情報を記録できる。

```text
送信元IPアドレス
宛先IPアドレス
送信元ポート番号
宛先ポート番号
プロトコル
通信量
通信が許可されたか、拒否されたか
```

通信結果は、代表的には以下のように記録される。

```text
ACCEPT
→ 通信が許可された

REJECT
→ 通信が拒否された
```

---

## 保存先

VPC Flow Logsの保存先として、主に以下を選択できる。

```text
CloudWatch Logs
S3
```

---

## どのような場面で使うのか

VPC Flow Logsは、主にトラブルシューティングや監査で利用する。

例として、Private EC2からRDSへ接続できない場合を考える。

```text
Private EC2
↓ TCP 3306
RDS for MySQL
```

接続できない原因として、Security GroupやNetwork ACLの設定ミスが考えられる。

VPC Flow Logsを確認すると、通信が許可されたのか、拒否されたのかを切り分けやすくなる。

```text
REJECT
↓
通信経路上で拒否されている可能性がある
↓
Security GroupやNetwork ACLなどを確認する
```

---

## 注意点

VPC Flow Logsは、通信の中身そのものを記録する機能ではない。

たとえば、Webサイトで送信した文章や、データベースへ実行したSQL文までは記録しない。

```text
記録するもの
→ どのIPアドレスから、どこへ通信したか
→ 何番ポートを利用したか
→ 通信が許可されたか、拒否されたか

記録しないもの
→ 通信の具体的な内容
→ SQL文
→ Webページの本文
→ パスワード
```

---

## VPNとは？

### 概要

VPN（Virtual Private Network）は、インターネットなどの共有ネットワーク上に、安全な通信経路を作る技術である。

一言で表すと、以下のような技術。

```text
VPN
=
インターネット上に安全な通信経路を作る技術
```

VPNを利用すると、離れた場所にあるネットワークやPCを、安全に接続できる。

```text
通信元
↓
暗号化された通信経路
↓
通信先
```

この暗号化された通信経路は、一般的にVPNトンネルと呼ばれる。

---

## AWSで利用される代表的な接続方法

AWSでは、接続元や目的に応じて複数の方法がある。

| 接続方法 | 接続するもの | 特徴 |
|---|---|---|
| Site-to-Site VPN | 会社のネットワークとAWS | 拠点全体をVPCへ接続する |
| Client VPN | 個人PCとAWS | 利用者のPCからVPCへ接続する |
| Direct Connect | 会社のネットワークとAWS | インターネットを経由しない専用接続を利用する |

---

## Site-to-Site VPNとは？

Site-to-Site VPNは、会社や自宅などのネットワークとAWSのVPCを接続する仕組みである。

```text
会社ネットワーク
↓
VPNトンネル
↓
AWS
↓
VPC
```

個人のPCを1台ずつ接続するのではなく、ネットワーク同士を接続する。

```text
会社ネットワーク
↕
AWSのVPC
```

---

## Client VPNとは？

Client VPNは、利用者のPCからAWSのVPCへ接続する仕組みである。

```text
個人PC
↓
VPN接続
↓
AWS
↓
VPC
```

リモートワーク中の利用者が、社内システムやAWS上のPrivate EC2へ接続する場合などに利用できる。

---

## Direct Connectとは？

Direct Connectは、自社ネットワークとAWSを専用の接続でつなぐサービスである。

```text
会社ネットワーク
↓
専用接続
↓
AWS
```

VPNとは異なり、インターネットを経由しない接続を利用する。

大量のデータを継続的に送受信したい場合や、安定した通信が必要な場合に利用される。

---

## Virtual Private Gatewayとは？

### 概要

Virtual Private Gatewayは、Site-to-Site VPNで利用されるAWS側の接続口である。

略して、`VGW` と呼ばれる。

```text
Virtual Private Gateway
=
AWS側に配置するVPNの入口
```

イメージは以下のとおり。

```text
会社ネットワーク
↓
VPNトンネル
↓
Virtual Private Gateway
↓
VPC
```

Virtual Private Gatewayは、VPCにアタッチして利用する。

---

## Customer Gatewayとは？

Customer Gatewayは、会社や自宅など、AWSの外側にあるネットワーク側のVPN接続口である。

略して、`CGW` と呼ばれる。

```text
Customer Gateway
=
自社側のVPN接続口
```

Site-to-Site VPNでは、AWS側と自社側の両方に接続口が必要になる。

```text
会社ネットワーク
↓
Customer Gateway
↓
VPNトンネル
↓
Virtual Private Gateway
↓
VPC
```

---

## Virtual Private GatewayとCustomer Gatewayの違い

| 種類 | 略称 | 配置する場所 | 役割 |
|---|---|---|---|
| Customer Gateway | CGW | 自社側・オンプレミス側 | 自社ネットワーク側のVPN接続口 |
| Virtual Private Gateway | VGW | AWS側 | VPC側のVPN接続口 |

セットで覚えると理解しやすい。

```text
Customer Gateway
→ 自社側のVPN接続口

Virtual Private Gateway
→ AWS側のVPN接続口
```

---

## 用語の注意点

Virtual Private Gatewayは、文脈によって以下のように呼ばれることがある。

```text
Virtual Private Gateway
仮想プライベートゲートウェイ
VGW
VPNゲートウェイ
```

ただし、AWSの正式な名称として覚える場合は、`Virtual Private Gateway` が基本となる。

---

## 全体像

会社のネットワークとAWSのVPCをSite-to-Site VPNで接続する場合、以下のような構成になる。

```text
会社ネットワーク
↓
Customer Gateway
↓
VPNトンネル
↓
Virtual Private Gateway
↓
VPC
↓
Private Subnet
↓
EC2・RDSなど
```

