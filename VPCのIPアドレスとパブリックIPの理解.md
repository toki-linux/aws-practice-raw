# VPCのIPアドレスとパブリックIPの理解

## 疑問

VPCでは `10.0.0.0/16` のようなプライベートIPアドレスを設定する。

しかし、

- 同じリージョンに他の利用者もいる
- 同じIPアドレス範囲を使っても良いと言われる

すると、

```text
みんなが同じ住所を使っていたら通信できないのでは？
```

という疑問が出てくる。

---

# 結論

VPC内のプライベートIPは、

```text
そのVPCの中だけで有効な住所
```

である。

そのため、

```text
自分のVPC
10.0.1.10

他人のVPC
10.0.1.10
```

が存在しても問題ない。

VPC同士は完全に分離されているためである。

---

# 家の例

例えば、

```text
Aさんの家
192.168.1.10

Bさんの家
192.168.1.10

会社
192.168.1.10
```

は普通に存在する。

同じ住所でも、

```text
どの家の中か
```

が違うため混乱しない。

VPCも同じ考え方である。

---

# 最初に持っていたイメージ

最初は次のようなイメージを持っていた。

```text
インターネット
↓
マンション全体の住所
↓
管理人
↓
各部屋
```

つまり、

```text
1つのグローバルIP
↓
管理人が振り分ける
```

というイメージ。

これは実は、

```text
家庭用ルーター
ポートフォワーディング
NAT
```

の考え方に近い。

---

# 家庭用ルーターの場合

例えば自宅では、

```text
グローバルIP
1.2.3.4
```

を家全体で共有している。

家の中には、

```text
PC
192.168.1.10

スマホ
192.168.1.20

タブレット
192.168.1.30
```

が存在する。

通信はルーターが管理する。

```text
インターネット
↓
グローバルIP
1.2.3.4
↓
ルーター
↓
各端末
```

これがポートフォワーディングやNATの考え方である。
ここ重要！！

---

# AWSのPublic EC2の場合

AWSでは少し考え方が異なる。

Public EC2は、

```text
EC2
├─ プライベートIP
│ 10.0.1.10
│
└─ パブリックIP
   54.x.x.x
```

のように2つの住所を持つ。

---

# 外から見た場合

```text
インターネット
↓
54.x.x.x
↓
EC2
```

---

# VPC内から見た場合

```text
10.0.1.10
```

で通信する。

つまり、

```text
外向け住所
内向け住所
```

の2つを持っている。

---

# 県と市の例

同じ場所でも、

```text
大阪府大阪市
```

と

```text
大阪市
```

のように、

見る人によって呼び方が変わる。

県外の人には、

```text
大阪府大阪市
```

が必要。

市内の人なら、

```text
大阪市
```

だけで伝わる。

EC2も同じである。

```text
外部
→ パブリックIP

VPC内部
→ プライベートIP
```

を使う。

---

# Internet Gatewayの役割

Internet Gatewayは、

```text
管理人
```

ではない。

イメージとしては、

```text
VPCとインターネットをつなぐ入口
```

である。

```text
54.x.x.x
↓
Internet Gateway
↓
10.0.1.10
```

という対応を利用して通信を届ける。

---

# Private EC2やRDSの場合

Private EC2やRDSは、

```text
プライベートIPのみ
```

を持つ。

例：

```text
Private EC2
10.0.2.10

RDS
10.0.3.20
```

そのため、

```text
インターネット
↓
直接アクセス
```

はできない。

---

# 実践で確認した構成

実際に構築した構成。

```text
PC
↓
Public EC2
(パブリックIPあり)
↓
Private EC2
(プライベートIPのみ)
↓
RDS
(プライベートIPのみ)
```

PCからは、

Public EC2のパブリックIPへ接続。

その後、

VPC内部ではプライベートIPで通信した。

---

# 学び

- VPCのIPアドレスは他人と重複しても問題ない
- VPCは完全に分離されたネットワーク空間
- Public EC2はパブリックIPとプライベートIPの両方を持つ
- Private EC2やRDSはプライベートIPのみ
- 最初にイメージしていたのはポートフォワーディングやNATルーターの考え方だった
- Internet Gatewayは管理人ではなく、インターネットとの接続口
- 同じリソースでも外部と内部で見える住所が異なる
