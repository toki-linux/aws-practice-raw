# AMI作成手順

## 目的

このドキュメントでは、EC2インスタンスを作成し、nginxとWebページを設定したうえで、その状態をAMIとして保存するまでの手順をまとめます。

今回の目的は、設定済みEC2をAMI化し、後から同じ状態のEC2を再作成できるようにすることです。

---

## 全体の流れ

```text
EC2インスタンス作成
↓
SSH接続
↓
nginxインストール
↓
index.html作成
↓
ブラウザで表示確認
↓
AMI作成
↓
AMIがavailableになることを確認
↓
スナップショット作成確認
```

---

## 1. EC2インスタンスを作成する

まず、AMI化する元になるEC2インスタンスを作成しました。

今回のEC2は、nginxをインストールしてWebページを表示できる状態にします。

設定内容は以下です。

```text
AMI：Ubuntu
インスタンスタイプ：低コスト系
キーペア：SSH接続用の既存キー
セキュリティグループ：SSHとHTTPを許可
```

---

## 2. セキュリティグループを設定する

今回の実践では、SSH接続とブラウザ表示確認を行うため、以下の通信を許可しました。

```text
SSH 22番
→ 自分のIPのみ許可

HTTP 80番
→ ブラウザ表示確認のため許可
```

今回の目的はWebページ表示確認も含むため、HTTP 80番を開放しました。

---

## 3. EC2へSSH接続する

作成したEC2インスタンスへSSH接続しました。

```bash
ssh -i キー名.pem ubuntu@パブリックIP
```

Ubuntu AMIの場合、ログインユーザーは基本的に `ubuntu` です。

---

## 4. パッケージ情報を更新する

nginxをインストールする前に、パッケージ情報を更新しました。

```bash
sudo apt update
```

---

## 5. nginxをインストールする

Webサーバとしてnginxをインストールしました。

```bash
sudo apt install -y nginx
```

---

## 6. nginxの起動状態を確認する

nginxが起動しているか確認しました。

```bash
sudo systemctl status nginx
```

nginxが `active (running)` になっていれば起動しています。

---

## 7. Webページを作成する

AMIから復元できたことが分かりやすいように、特徴のある内容で `index.html` を作成しました。

```bash
echo "Hello from AMI practice instance" | sudo tee /var/www/html/index.html
```

作成したファイルを確認します。

```bash
cat /var/www/html/index.html
```

---

## 8. EC2内でWebページを確認する

EC2内からnginxの表示を確認しました。

```bash
curl localhost
```

期待する表示です。

```text
Hello from AMI practice instance
```

---

## 9. ブラウザでWebページを確認する

ブラウザで以下にアクセスしました。

```text
http://元EC2のパブリックIP
```

作成したWebページが表示されることを確認しました。

---

## 10. AMIを作成する

Webページ表示まで確認できたら、元EC2からAMIを作成しました。

操作手順は以下です。

```text
EC2
→ インスタンス
→ 対象インスタンスを選択
→ アクション
→ イメージとテンプレート
→ イメージを作成
```

設定例です。

```text
イメージ名：
ami-nginx-practice

イメージの説明：
nginx installed and custom index.html created for AMI practice
```

---

## 11. 「インスタンスを再起動」はチェックありにする

AMI作成画面で「インスタンスを再起動」のチェックがありました。

今回はデフォルトのまま、チェックありで進めました。

```text
インスタンスを再起動：チェックあり
```

理由は、AMI作成時にインスタンスを再起動することで、ディスクの状態を整えてからスナップショットを取得しやすくなるためです。

今回の環境は学習用で、一時的に停止しても問題ありません。

```text
学習用環境
停止しても問題なし
データ整合性を優先

→ チェックありで作成
```

---

## 12. AMI作成状況を確認する

AMIの作成状況を確認しました。

確認場所です。

```text
EC2
→ イメージ
→ AMI
```

作成直後は以下のような状態になることがあります。

```text
pending
```

しばらく待って、以下になればAMI作成完了です。

```text
available
```

---

## 13. スナップショットを確認する

AMIを作成すると、裏側でEBSスナップショットも作成されます。

確認場所です。

```text
EC2
→ Elastic Block Store
→ スナップショット
```

AMIに関連するスナップショットが作成されていることを確認しました。

このスナップショットは、後片付け時に削除対象になります。

---

## ここまででできたこと

```text
元EC2を作成した
nginxをインストールした
index.htmlを作成した
ブラウザでWebページを表示した
元EC2からAMIを作成した
AMIがavailableになることを確認した
AMI作成に伴うスナップショットを確認した
```

---

## 次に行うこと

次は、作成したAMIから新しいEC2インスタンスを起動し、元EC2と同じWebページが表示されるか確認します。
