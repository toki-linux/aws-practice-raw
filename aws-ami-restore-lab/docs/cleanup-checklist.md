# 後片付けチェックリスト：AMI実践

## 目的

このドキュメントでは、AMI作成・AMIからEC2再作成の実践後に削除したリソースと、確認すべきポイントを整理します。

AMI実践では、EC2だけでなくAMIとスナップショットも確認する必要があります。

---

## 削除対象

今回の実践後に削除・確認したものは以下です。

```text
元EC2インスタンス
AMIから作成した新EC2インスタンス
作成したAMI
関連スナップショット
EBSボリューム
Elastic IP
```

---

## 1. 元EC2インスタンスを終了する

まず、AMI作成元になったEC2インスタンスを終了しました。

操作です。

```text
EC2
→ インスタンス
→ 元EC2を選択
→ インスタンスの状態
→ インスタンスを終了
```

注意点です。

```text
停止
→ インスタンスは残る

終了
→ インスタンスを削除する
```

課金を抑えるため、停止ではなく終了を選びました。

---

## 2. AMIから作成した新EC2インスタンスを終了する

次に、AMIから起動した新EC2インスタンスも終了しました。

操作です。

```text
EC2
→ インスタンス
→ AMIから作成した新EC2を選択
→ インスタンスの状態
→ インスタンスを終了
```

今回の確認は完了しているため、新EC2も削除しました。

---

## 3. AMIを登録解除する

次に、作成したAMIを登録解除しました。

操作です。

```text
EC2
→ イメージ
→ AMI
→ 作成したAMIを選択
→ アクション
→ AMIを登録解除
```

AMIの削除は、正確には「登録解除」です。

```text
AMIを登録解除
→ EC2起動元として使えないようにする
```

---

## 4. 関連スナップショットを削除する

AMIを作成すると、裏側でEBSスナップショットが作成されます。

AMIを登録解除しても、スナップショットが残ることがあります。

そのため、AMI登録解除後にスナップショットを確認しました。

操作です。

```text
EC2
→ Elastic Block Store
→ スナップショット
→ 今回のAMIに関連するスナップショットを選択
→ 削除
```

重要ポイントです。

```text
AMIだけ消しても、スナップショットが残る場合がある
```

課金を防ぐため、スナップショットも確認して削除しました。

---

## 5. EBSボリュームを確認する

EC2終了後、不要なEBSボリュームが残っていないか確認しました。

確認場所です。

```text
EC2
→ Elastic Block Store
→ ボリューム
```

確認することです。

```text
元EC2に関連するEBSが残っていないか
新EC2に関連するEBSが残っていないか
不要なボリュームがavailableで残っていないか
```

不要なEBSが残っている場合は削除します。

---

## 6. Elastic IPを確認する

今回Elastic IPは作成していません。

そのため、削除対応は不要でした。

もしElastic IPを作成していた場合は、未使用のElastic IPが残っていないか確認します。

```text
EC2
→ Elastic IP
```

未使用のElastic IPは課金につながる可能性があるため、不要なら解放します。

---

## 7. セキュリティグループについて

セキュリティグループは、基本的に残しておいても課金対象ではありません。

そのため、課金面では残しておいて問題ありません。

```text
課金面
→ 残してOK

管理面
→ 不要なら削除してOK
```

練習用のセキュリティグループが増えてきた場合は、整理します。

---

## 8. 削除後の確認

今回の削除確認は以下です。

```text
元EC2インスタンス：削除済み
AMIから作成した新EC2インスタンス：削除済み
AMI：登録解除済み
関連スナップショット：削除済み
不要EBSボリューム：残りなし
Elastic IP：作成していない
セキュリティグループ：課金対象ではないため残してOK
```

---

## 9. AMI実践で特に注意する削除対象

AMI実践では、特に以下を忘れないようにします。

```text
AMI
スナップショット
```

理由です。

```text
AMIを作成すると、裏側でスナップショットが作成される
AMIを登録解除しても、スナップショットが残ることがある
```

そのため、削除時は以下の順で確認します。

```text
AMIを登録解除
↓
関連スナップショットを削除
```

---

## 10. AWS学習後に確認すべき課金対象

AWS学習後に特に確認するものです。

```text
EC2 実行中インスタンス
EBS ボリューム
スナップショット
Elastic IP
NAT Gateway
Load Balancer
RDS
```

今回の範囲では、特に以下を確認しました。

```text
EC2が終了しているか
AMIを登録解除したか
スナップショットを削除したか
EBSが残っていないか
Elastic IPを作っていないか
```

---

## 後片付けの基本方針

AWS学習では、作ることと削除することをセットにします。

```text
作る
↓
試す
↓
確認する
↓
削除する
```

特にAMI実践では、EC2だけでなくAMIとスナップショットまで削除確認することが重要です。

---

## まとめ

今回のAMI実践では、元EC2からAMIを作成し、AMIから新EC2を起動してWebページ表示を確認しました。

その後、課金を抑えるために以下を削除しました。

```text
元EC2
新EC2
AMI
スナップショット
不要EBS
```

AMI実践では、AMIの登録解除だけでなく、関連スナップショットの削除確認まで行うことが大切です。
