SSHは自分のIPのみアクセス可能に設定できたが、HTTPは0.0.0.0に設定できなかった。なぜか？推測

EC2作成画面で「HTTPトラフィックを許可」にチェックを入れた。
この設定により、HTTP用の80番ポートが許可され、ブラウザからパブリックIPv4アドレスでWebページを表示できた。

次回は、作成後のセキュリティグループ詳細画面で、HTTPルールのソースが `0.0.0.0/0` になっていることを確認する。

後日
EC2作成時にセキュリティグループを設定し、SSH接続用に22番ポート、Webアクセス用にHTTP 80番ポートを許可した。

SSHは自分のIPからのみ許可し、HTTPはブラウザからアクセスできるように許可した。
その後、EC2上にnginxを導入し、パブリックIPv4アドレスからWebページを表示できることを確認した。

チェックを入れただけでも、裏側では
HTTP / TCP / 80 / 0.0.0.0/0
というルールが作られている。
