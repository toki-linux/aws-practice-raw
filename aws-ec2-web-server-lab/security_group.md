# セキュリティグループ設定

## 概要

EC2インスタンスに対する通信を許可するため、セキュリティグループを設定した。

セキュリティグループは、EC2に対する仮想ファイアウォールのような役割を持つ。

## 設定したインバウンドルール

| 種類 | ポート | 送信元 | 用途 |
|---|---:|---|---|
| SSH | 22 | 自分のIP | EC2へSSH接続するため |
| HTTP | 80 | 0.0.0.0/0 | ブラウザからWebページを見るため |

## SSH 22番

SSH接続には22番ポートを使用する。

```text
SSH 22
→ EC2にログインして操作するための入口
```

SSHは誰でもアクセスできる状態にすると危険なため、送信元は自分のIPに限定した。

## HTTP 80番

ブラウザからWebページを表示するには80番ポートを使用する。

```text
HTTP 80
→ Webページを表示するための入口
```

今回はブラウザからアクセスできるように、HTTPは `0.0.0.0/0` を指定した。

## 接続できない場合の確認ポイント

SSH接続できない場合は、以下を確認する。

- インスタンスが実行中か
- パブリックIPv4アドレスが正しいか
- セキュリティグループで22番ポートが許可されているか
- 秘密鍵が正しいか
- SSHユーザー名が `ubuntu` になっているか

Webページが表示されない場合は、以下を確認する。

- nginxが起動しているか
- セキュリティグループで80番ポートが許可されているか
- URLが `https` ではなく `http` になっているか
- パブリックIPv4アドレスでアクセスしているか

## 学んだこと

セキュリティグループでポートを許可しないと、EC2の中でサービスが起動していても外部からアクセスできない。

そのため、サーバ内部の設定だけでなく、AWS側の通信制御も確認する必要がある。
