# 後片付けチェックリスト：EC2 + IAM Role + S3

## 目的

このドキュメントでは、EC2 + IAM Role + S3の実践後に削除したリソースと、確認すべきポイントを整理する。

課金を抑え、不要なリソースを残さないために、実践後は必ず後片付けを行う。

---

## 削除対象

今回削除したものは以下。

```text
EC2インスタンス
S3内のtest.txt
S3バケット
IAM Role
```

---

## 1. EC2インスタンスを終了する

### 操作

```text
EC2
→ インスタンス
→ 今回作成したインスタンスを選択
→ インスタンスの状態
→ インスタンスを終了
```

### 注意点

「停止」ではなく「終了」を選ぶ。

```text
停止
→ インスタンスは残る

終了
→ インスタンスを削除する
```

---

## 2. S3内のファイルを削除する

S3バケットを削除する前に、中のオブジェクトを削除する。

### 操作

```text
S3
→ 今回作成したバケットを開く
→ test.txtを選択
→ 削除
```

S3バケットは、中身が残っていると削除できない場合がある。

---

## 3. S3バケットを削除する

S3内のファイルを削除した後、バケットを削除する。

### 操作

```text
S3
→ バケット一覧
→ 今回作成したバケットを選択
→ 削除
```

確認画面でバケット名の入力が求められた場合は、表示されたバケット名をそのまま入力する。

---

## 4. IAM Roleを削除する

今回作成した練習用IAM Roleを削除する。

### 操作

```text
IAM
→ ロール
→ 今回作成したRoleを選択
→ 削除
```

### 補足

IAM Role自体は基本的に課金対象ではない。

ただし、練習用で今後使わない場合は削除しておくと、管理しやすい。

---

## 5. 削除後の確認

削除後、以下を確認する。

```text
EC2
→ 実行中のインスタンスがない

S3
→ 今回作成したバケットがない

IAM
→ 今回作成したRoleがない
```

---

## 6. Elastic IPの確認

今回Elastic IPを作成していない場合は不要。

もしElastic IPを作成していた場合は、未使用のElastic IPが残っていないか確認する。

```text
EC2
→ Elastic IP
→ 未使用のElastic IPがないか確認
```

Elastic IPは、使っていない状態で残すと課金につながる可能性があるため注意する。

---

## 7. EBSボリュームの確認

EC2終了時にルートEBSも削除される設定になっていれば、基本的に追加対応は不要。

ただし、念のため以下を確認する。

```text
EC2
→ ボリューム
→ 今回のインスタンスに関連する不要なEBSが残っていないか確認
```

---

## 8. スナップショットの確認

今回スナップショットを作成していない場合は不要。

もし作成していた場合は、不要なスナップショットが残っていないか確認する。

```text
EC2
→ スナップショット
→ 不要なスナップショットがないか確認
```

---

## 今回の削除確認メモ

```text
EC2インスタンス：削除済み
S3内のtest.txt：削除済み
S3バケット：削除済み
IAM Role：削除済み
Elastic IP：作成していない
スナップショット：作成していない
```

---

## 後片付けで大事なこと

AWS学習では、作ることと同じくらい、消すことも大事。

特に練習環境では、使い終わったらすぐ削除する。

```text
作る
↓
試す
↓
確認する
↓
消す
```

この流れを毎回セットにすることで、不要な課金を防ぎながら実践を続けられる。
