# セキュリティ面のポイント：EC2 + IAM Role + S3

## 目的

このドキュメントでは、EC2からIAM Roleを使ってS3へアクセスする実践で意識したセキュリティ面のポイントを整理する。

---

## 1. アクセスキーをEC2内に置かない

今回の一番重要なポイントは、EC2内にアクセスキーを保存しなかったこと。

```text
やらないこと：
EC2内でaws configureを使ってアクセスキーを保存する

やったこと：
EC2にIAM Roleを付けてAWS CLIを使う
```

アクセスキーをサーバ内に置くと、漏えいしたときのリスクがある。

IAM Roleを使うことで、EC2に一時的な認証情報を渡し、アクセスキーを直接管理しなくて済む。

---

## 2. IAM Roleで必要な権限だけを渡す

今回のIAM Roleには、S3読み取り権限のみを付与した。

使ったポリシー。

```text
AmazonS3ReadOnlyAccess
```

今回必要だった操作。

```text
S3バケット一覧を見る
S3バケット内のファイルを見る
S3からファイルを取得する
```

不要だった操作。

```text
S3へアップロードする
S3上のファイルを削除する
S3バケットを削除する
```

そのため、書き込みや削除の権限は付けなかった。

---

## 3. S3バケットを公開しない

今回のS3バケットは、パブリックアクセスをすべてブロックした。

理由は、今回の目的がS3のWeb公開ではなく、EC2からS3を読むことだったため。

```text
今回の目的：
EC2からS3へアクセスする

今回不要なこと：
インターネット上にS3を公開する
```

不要な公開設定はしない。

---

## 4. セキュリティグループはSSHのみ許可

今回のEC2では、セキュリティグループでSSHのみ許可した。

```text
SSH 22番：許可
HTTP 80番：許可しない
```

理由は、今回Webサーバを立てる実践ではなかったため。

必要な通信だけを許可することで、外部に公開する範囲を減らした。

---

## 5. HTTPを開けなかった理由

前回までのEC2 Webサーバ実践ではHTTP 80番を開けた。

しかし今回は、nginxを使わず、ブラウザ表示もしない。

目的はIAM RoleとS3アクセスの確認。

そのため、HTTP 80番は不要。

```text
必要：SSH接続
不要：HTTPアクセス
```

---

## 6. IAM Roleの信頼されたエンティティをEC2にする

IAM Roleでは、誰がそのRoleを使えるかを設定する。

今回のRoleを使うのはEC2なので、信頼されたエンティティはEC2にした。

```text
信頼されたエンティティ：EC2
```

これにより、EC2インスタンスがこのRoleを引き受けられるようになる。

---

## 7. aws configureをしないこと自体が確認ポイント

今回 `aws configure` をしなかったことは重要。

なぜなら、`aws configure` を使ってアクセスキーを入れてしまうと、IAM Roleで動いているのか、アクセスキーで動いているのか分かりにくくなるため。

今回確認したかった状態。

```text
EC2にIAM Roleが付いている
↓
aws configureはしていない
↓
それでもAWS CLIでS3にアクセスできる
```

この状態を確認することで、IAM Roleの効果を理解できる。

---

## 8. 後片付けをする

実践後は、作成したリソースを削除した。

```text
EC2インスタンス
S3内のtest.txt
S3バケット
IAM Role
```

理由は、不要なリソースを残さないため。

特にEC2は起動していると課金対象になるため、練習後は終了する。

S3バケットも練習用なので削除した。

IAM Role自体は基本的に課金対象ではないが、不要なRoleを残さないため削除した。

---

## 今回のセキュリティ方針

今回の設定では、以下を意識した。

```text
アクセスキーを置かない
必要な権限だけ付ける
S3を公開しない
必要なポートだけ開ける
使い終わったリソースは削除する
```

---

## まとめ

今回の実践で重要なのは、AWSでは「権限をどう渡すか」が大事だということ。

EC2からAWSサービスを使いたい場合、アクセスキーを直接置くのではなく、IAM Roleを使う。

```text
悪い例：
EC2内にアクセスキーを保存する

良い例：
EC2にIAM Roleを付ける
```

IAM Roleを使うことで、安全にAWSサービスへアクセスできる。
