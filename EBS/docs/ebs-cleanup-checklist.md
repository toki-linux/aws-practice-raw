# EBS追加ボリューム実践：後片付けチェックリスト

## 目的

EBS追加ボリューム実践後に、不要なリソースを削除し、課金を抑えるためのチェックリスト。

---

## 削除対象

今回の実践後に確認・削除したもの。

```text
EC2インスタンス
追加EBSボリューム
スナップショット
Elastic IP
```

---

## 1. EC2インスタンスを終了する

### 操作

```text
EC2
→ インスタンス
→ 今回作成したインスタンスを選択
→ インスタンスの状態
→ インスタンスを終了
```

### 注意点

「停止」ではなく「終了」を選ぶ。

```text
停止
→ インスタンスは残る

終了
→ インスタンスを削除する
```

---

## 2. 追加EBSが残っていないか確認する

EC2終了後、追加EBSが自動削除されずに残っている場合がある。

そのため、ボリューム一覧を確認する。

```text
EC2
→ Elastic Block Store
→ ボリューム
```

今回作成した1GiBの追加EBSが残っていないか確認する。

---

## 3. 追加EBSが残っていたら削除する

追加EBSが残っていた場合は、手動で削除する。

操作。

```text
EC2
→ Elastic Block Store
→ ボリューム
→ 追加EBSを選択
→ アクション
→ ボリュームを削除
```

今回の実践では、追加EBSも削除した。

---

## 4. スナップショットを確認する

今回スナップショットは作成していない。

そのため、基本的には削除不要。

ただし、作成した場合は以下を確認する。

```text
EC2
→ Elastic Block Store
→ スナップショット
```

不要なスナップショットがあれば削除する。

---

## 5. Elastic IPを確認する

今回Elastic IPは作成していない。

そのため、削除不要。

もしElastic IPを作成していた場合は、未使用のElastic IPが残っていないか確認する。

```text
EC2
→ Elastic IP
```

未使用のElastic IPがある場合は、解放する。

---

## 6. セキュリティグループについて

セキュリティグループは、残しておいても基本的に課金対象ではない。

そのため、課金面では残しておいて問題ない。

```text
課金面
→ 残してOK

管理面
→ 不要なら削除してOK
```

今後もSSH用のセキュリティグループとして使う場合は残してよい。

練習用のセキュリティグループが増えてきた場合は、整理する。

---

## 7. 削除後の確認

今回の削除確認。

```text
EC2インスタンス：終了済み
追加EBSボリューム：削除済み
スナップショット：作成していない
Elastic IP：作成していない
セキュリティグループ：残してOK
```

## 課金対策として特に見るもの

AWS学習後に確認すべき主な課金対象。

```text
EC2 実行中インスタンス
EBS ボリューム
Elastic IP
スナップショット
NAT Gateway
Load Balancer
RDS
```

今回の範囲では、特に以下を見る。

```text
EC2が終了しているか
追加EBSが残っていないか
Elastic IPを作っていないか
スナップショットを作っていないか
```

---

## 後片付けの基本方針

AWS学習では、作成と削除をセットにする。

```text
作る
↓
試す
↓
確認する
↓
削除する
```

課金を抑えながら実践するため、使い終わったリソースはすぐに削除する。

---

## 今回のまとめ

今回のEBS実践では、EC2と追加EBSを作成し、Ubuntu上で `/data` にマウントして使えることを確認した。

その後、課金を抑えるためにEC2と追加EBSを削除した。

セキュリティグループは課金対象ではないため、必要に応じて残す。
