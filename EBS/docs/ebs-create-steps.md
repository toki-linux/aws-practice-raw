# EBS追加ボリューム作成手順

## 目的

EC2インスタンスに追加EBSボリュームを作成・アタッチし、Ubuntu上でデータ保存用ディスクとして使えるようにする。

今回のゴールは以下。

```text
追加EBSを作成する
↓
EC2にアタッチする
↓
Ubuntu側で認識する
↓
/data にマウントして使えるようにする
↓
再起動後も自動マウントされるようにする
```

---

## 前提理解

EBSは、EC2に接続して使うブロックストレージサービス。

EC2を作成すると、通常はOS用のルートEBSが最初から付いている。

```text
ルートEBS
→ OSが入っているディスク
→ / として使われる

追加EBS
→ 後から追加するデータ保存用ディスク
→ 今回は /data として使う
```

今回やることは、OS用ディスクとは別に、データ保存用のEBSを追加すること。

---

## 構成イメージ

```text
EC2
├── ルートEBS
│   └── /
│       ├── /home
│       ├── /etc
│       └── /var
│
└── 追加EBS
    └── /data
```

---

## 1. EC2インスタンスを起動する

まず、EC2インスタンスを1台起動した。

今回の目的はEBS追加ボリュームの実践なので、Webサーバは不要。

設定は以下。

```text
AMI：Ubuntu
インスタンスタイプ：低コスト系
セキュリティグループ：SSHのみ許可
HTTP：不要なので開放しない
```

---

## 2. EC2のアベイラビリティゾーンを確認する

EBSは、同じアベイラビリティゾーンのEC2にしかアタッチできない。

そのため、先にEC2インスタンスのアベイラビリティゾーンを確認した。

確認場所。

```text
EC2
→ インスタンス
→ 対象インスタンスを選択
→ 詳細
→ アベイラビリティーゾーン
```

例。

```text
ap-northeast-1a
```

---

## 3. EBSボリュームを作成する

EC2画面の左メニューから、EBSボリュームを作成した。

操作。

```text
EC2
→ Elastic Block Store
→ ボリューム
→ ボリュームを作成
```

設定内容。

```text
ボリュームタイプ：gp3
サイズ：1 GiB
アベイラビリティーゾーン：EC2と同じAZ
スナップショットID：なし
暗号化：デフォルト
```

タグを付ける場合。

```text
キー：Name
値：ebs-practice-volume
```

---

## 4. なぜ1GiBにしたか

今回の目的は、EBSの作成・アタッチ・マウント・自動マウントを確認すること。

大量の容量は不要なので、課金を抑えるために1GiBで作成した。

```text
目的：操作理解
必要容量：小さくてよい
設定：1GiB
```

---

## 5. なぜEC2と同じAZにしたか

EBSは、同じアベイラビリティゾーン内のEC2インスタンスにアタッチする。

そのため、EC2と追加EBSのAZを合わせる必要がある。

```text
EC2：ap-northeast-1a
EBS：ap-northeast-1a
```

このように、EC2とEBSのAZを一致させた。

---

## 6. EBSボリュームをEC2にアタッチする

作成したEBSボリュームが `available` になったら、EC2にアタッチした。

操作。

```text
EC2
→ Elastic Block Store
→ ボリューム
→ 作成したボリュームを選択
→ アクション
→ ボリュームをアタッチ
```

設定内容。

```text
インスタンス：今回作成したEC2
デバイス名：/dev/sdf
```

---

## 7. アタッチ後の状態

EBSをEC2にアタッチすると、EC2側に新しいディスクとして接続される。

ただし、この時点ではまだUbuntu上で普通の保存場所としては使えない。

```text
EBSをアタッチした
→ EC2にディスクは接続された

でも、まだマウントしていない
→ Linux上のディレクトリとしては使えない
```

---

## 8. ここまでで完了したこと

```text
EC2を起動した
EC2のAZを確認した
EC2と同じAZにEBSを作成した
1GiBの追加EBSを作成した
追加EBSをEC2にアタッチした
```

この後、Ubuntu側で以下を行う。

```text
lsblkで追加EBSを確認する
ファイルシステムを作成する
/dataにマウントする
自動マウント設定をする
```
